# Project One – Business Requirements Document

This folder contains the Business Requirements Document (BRD) for the DriverPass scheduling and training system, developed as part of CS 255: System Analysis and Design.

## File Included

- `DriverPass_BRD.docx` — Finalized BRD including system goals, requirements, user interface design, and embedded Gantt chart.

## Key Features

- Clear system objectives and measurable goals  
- Functional and nonfunctional requirements  
- Role-based user interface design  
- System assumptions and limitations  
- Gantt chart showing a six-week development timeline

## Tools Used

- Microsoft Word — Document creation and formatting  
- Draw.io — Gantt chart creation

---

Created for **CS 255: System Analysis and Design** at Southern New Hampshire University.
